XXE attack via modified content type 

Most POST requests use a default content type that is generated by HTML forms,
 such as application/x-www-form-urlencoded. 
Some web sites expect to receive requests in this format but will process other content types, including XML.

POST /hackingman HTTP/1.0
Content-Type: application/x-www-form-urlencoded
Content-Length: 7cicada=3301


and you submit the same request in XML format:

POST /hackingman HTTP/1.0
Content-Type: text/html
Content-Length: 7<?xml version="1.0" encoding="UTF-8"><cicada>3301</cicada>

and the server accepts this then you have just found another surface to inject some malicious XML.
